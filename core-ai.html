Deep Dive — Architecture

⚡ The Transformer Architecture

The 2017 invention that powers everything — GPT-4, Claude, Gemini, Llama. Understanding the Transformer gives you the mental model for all of modern AI.

📖 This page explains the Transformer introduced in "Attention Is All You Need" (Vaswani et al., 2017) — one of the most cited papers in the history of computer science.

The Problem Transformers Solved

Before Transformers: Sequential Reading

Previous architectures — RNNs, LSTMs — processed text sequentially, like reading left to right. This created two fatal problems: vanishing gradients (information from early in a sequence dissolves over many steps, making long-range dependencies impossible to learn) and no parallelism (token 100 can only be processed after tokens 1–99, preventing efficient GPU utilisation).

The result: these models struggled with long documents and were slow to train. The field needed a fundamentally different approach.

The Transformer's Solution: Attention Over Everything

The Transformer discards sequential processing entirely. Instead, every token attends to every other token in the sequence simultaneously. The model processes all positions in parallel and uses attention weights to determine how much each position should influence the representation of every other position.

Result: long-range dependencies are learned as easily as short-range ones. The word at position 1 can directly influence the representation at position 500, with no information loss. And parallel processing of all positions means GPUs — which excel at parallel operations — can be fully utilised.

The Architecture Step by Step

1

Input Tokenisation

Your text is split into tokens using a learned vocabulary (e.g., 50,000 items for GPT). "Hello, world!" → ["Hello", ",", " world", "!"] → [15496, 11, 995, 0] (integer IDs). These IDs are the model's actual input.

Tokenisation algorithms like Byte-Pair Encoding (BPE) balance vocabulary size against coverage of all possible text, including code, maths, and multilingual content.

2

Token Embeddings

Each integer ID is looked up in an embedding matrix (dimensions: vocab_size × d_model). This produces a dense vector for each token — e.g., a 4,096-dimensional vector for GPT-3 large. These vectors encode semantic meaning and are the starting representation of each token.

At the start of training, embeddings are random. Through backpropagation, they organise themselves so that semantically similar tokens end up close together in this high-dimensional space.

3

Positional Encoding

Attention is order-agnostic — without extra information, the model can't distinguish "dog bites man" from "man bites dog." Positional encodings add order information. The original paper used sinusoidal functions; modern models learn positional encodings, or use rotary position embeddings (RoPE) that scale to long contexts more gracefully.

The final input to the first transformer layer is: embedding + positional_encoding — a vector containing both meaning and position.

4

Multi-Head Self-Attention (The Core)

For each token, three projections are computed from the embedding: Query (Q) — "what am I looking for?", Key (K) — "what do I contain?", Value (V) — "what information do I provide?"

Attention score between token i and token j = softmax(Q_i · K_j / √d_k). These scores become attention weights — a probability distribution over all positions. The output for position i is a weighted sum of all Value vectors: output_i = Σ weight_{ij} × V_j.

Multi-head: This is done h times in parallel with different weight matrices (typically h=8 to 96 heads). Each head can specialise in different relationship types. Results are concatenated and linearly projected.

5

Add & Norm (Residual Connection)

The output of attention is added back to the original input (residual/skip connection): output = LayerNorm(x + Attention(x)). This serves two purposes: it preserves gradient flow (preventing vanishing gradients in deep networks) and allows the network to learn "adjustments" rather than complete transformations — making training much more stable.

6

Feed-Forward Network (FFN)

After attention, each token's representation passes through an identical two-layer MLP applied position-wise: FFN(x) = max(0, xW₁ + b₁)W₂ + b₂. The intermediate dimension is typically 4× the model dimension (e.g., 4,096 → 16,384 → 4,096 for GPT-3 base).

The FFN adds computation capacity beyond what attention provides. Recent research suggests the FFN layers act as "key-value memories" storing factual associations learned during training. Another residual connection + layer norm follows.

7

Stack N Layers

Steps 4–6 form one "transformer block." These blocks are stacked: GPT-2 small uses 12 layers; GPT-3 uses 96; GPT-4 is rumoured to use 120+ layers. Each layer refines representations. Lower layers learn syntactic features (grammar, word order). Higher layers learn semantic features (reasoning, facts, pragmatics).

This stacking is where "deep" learning lives — not the depth of individual operations, but the depth of stacked transformation stages.

8

Output: Predicting the Next Token

The final layer's output for the last token position is projected through a linear layer (d_model → vocab_size) followed by softmax, producing a probability distribution over all ~50,000 possible next tokens.

The model samples from this distribution (controlled by temperature) to select the next token, appends it to the context, and runs another forward pass. This autoregressive loop generates text one token at a time.

Architecture Variants

📝

Decoder-only (GPT family)

Uses causal (masked) self-attention — each token only attends to previous tokens. Used for text generation. GPT-4, Claude, Gemini, Llama all use decoder-only architecture.

🔄

Encoder-Decoder (T5, BART)

Original Transformer design. Encoder processes input with full bidirectional attention; decoder generates output attending to encoder. Best for translation and summarisation.

👁️

Encoder-only (BERT)

Bidirectional attention over the full input. Can't generate text. Used for classification, NER, embeddings. Fine-tunes efficiently on small datasets.

🖼️

Vision Transformer (ViT)

Images split into 16×16 patches, each treated as a token. Same transformer architecture applied to visual data. Now the dominant architecture for image tasks.

🧩

Mixture of Experts (MoE)

Multiple specialist "expert" FFN networks. A routing network selects 2–4 experts per token. Used in Mixtral, GPT-4 (speculated), and Gemini. Same compute as dense model but more parameters.

⚡

Flash Attention

An algorithm optimisation (not architecture change) that makes attention computation 2–4x faster with less memory by fusing operations and using hardware-aware tiling. Now standard.

Deep Dive — Fundamentals

🕸️ Neural Networks — From Neurons to Depth

Understanding neural networks is understanding the foundation of all modern AI. This section builds from a single artificial neuron to the 100-billion parameter systems running today.

The Biological Inspiration

The brain contains ~86 billion neurons. Each neuron receives signals from thousands of other neurons through dendrites, accumulates these signals, and if the total exceeds a threshold, fires an output signal down its axon to the next neurons. Neural networks are a loose mathematical model of this process — not a literal simulation of the brain, but inspired by its architecture.

A single artificial neuron: takes inputs x₁, x₂, ... xₙ, multiplies each by a learned weight w₁, w₂, ... wₙ, sums them up (plus a bias term), and passes the result through a non-linear activation function. Simple. Chain thousands of these together and stack them in layers, and you get a neural network capable of approximating any continuous function.

Layers and Depth

Input Layer

Raw Data Entry

The input layer receives raw data — pixel values, token embeddings, audio spectrograms. No transformation happens here; it simply passes data to the first hidden layer. The number of input neurons matches the data dimensionality.

Hidden Layers

Where Learning Happens

Hidden layers learn increasingly abstract representations. Layer 1 might learn edges in images. Layer 5 learns textures. Layer 20 learns object parts. Layer 50 learns entire objects in context. The more layers (depth), the more complex the learned hierarchy.

Output Layer

Final Prediction

The output layer produces the final result. For classification: N neurons (one per class), softmax activation to get probabilities. For generation: vocab_size neurons, each representing the probability of that token being next.

Activation Functions

The Non-linearity

Without activation functions, any number of linear layers collapses to a single linear function. ReLU (max(0,x)) is the standard: simple, efficient, doesn't suffer from vanishing gradients like sigmoid. Variants: LeakyReLU, GELU (used in transformers), SiLU.

Backpropagation — How Networks Learn

The Learning Algorithm

Forward pass: Input flows through the network, layer by layer, to produce a prediction. Compare the prediction to the true label using a loss function (e.g., cross-entropy for classification). The loss is a single number measuring how wrong the prediction was.

Backward pass: Using calculus (the chain rule), compute the gradient of the loss with respect to every weight in the network — how much should each weight change to reduce the loss? This computation flows backwards from the output to the input (hence "backpropagation").

Weight update: Each weight is adjusted by a small step in the direction of the negative gradient: w ← w - learning_rate × ∂L/∂w. This is Stochastic Gradient Descent (SGD) or its variants (Adam, AdamW). Repeat millions of times on mini-batches of training data, and the weights converge toward values that minimise loss on the training set.

Forward Pass:
  Input → Layer 1 → Layer 2 → ... → Output → Loss = 2.7

Backward Pass (chain rule):
  ∂Loss/∂w_last = -0.3   ← how much does last layer's weight affect loss?
  ∂Loss/∂w_prev = -0.1   ← propagate gradient backwards through layers
  ∂Loss/∂w_first = +0.05 ← gradient at first layer

Weight Update (lr=0.01):
  w_last  ← w_last  - 0.01 × (-0.3) = w_last  + 0.003  ✓ moves toward lower loss
  w_prev  ← w_prev  - 0.01 × (-0.1) = w_prev  + 0.001
  w_first ← w_first - 0.01 × (+0.05)= w_first - 0.0005

Key Training Concepts

Problem

Vanishing Gradients

In deep networks, gradients can become exponentially small as they propagate backwards, making early layers train extremely slowly. Solved by: ReLU activations, residual connections, batch normalisation, careful initialisation.

Problem

Overfitting

The model memorises training data rather than learning generalisable patterns. Solutions: dropout (randomly deactivating neurons during training), weight decay (penalising large weights), data augmentation, early stopping.

Technique

Batch Normalisation

Normalise layer activations across the training mini-batch. Stabilises training, allows higher learning rates, and acts as regularisation. Transformers use Layer Normalisation (normalise across features, not batch) instead.

Technique

Transfer Learning

Take a network trained on task A (e.g., ImageNet classification), freeze the early layers (generic features), and fine-tune the later layers on task B (e.g., medical imaging). Requires dramatically less data for task B. Foundation of all modern LLM applications.

💡 The Universal Approximation Theorem states that a sufficiently wide single hidden-layer neural network can approximate any continuous function to arbitrary precision. Depth (multiple layers) provides efficiency — deep networks can represent functions exponentially more compactly than shallow ones. This mathematical fact underpins all of deep learning's expressive power.
Deep Dive — Process

🏋️ How LLMs Are Trained

From raw internet text to a helpful AI assistant — the complete training pipeline that creates models like Claude, GPT-4, and Gemini.

💰 Training a frontier LLM is one of the most expensive computational endeavours in history. GPT-4 reportedly cost over $100M to train. Claude 3 Opus training costs are similarly massive. This section explains why — and every step that justifies it.

Phase 1: Data Collection & Curation

Building the Training Corpus

Modern LLMs train on datasets measured in trillions of tokens (roughly equivalent to millions of books). The primary source is web text — Common Crawl, a massive periodic scrape of the public web, contains petabytes of text. This is supplemented with curated sources: books (BookCorpus, Project Gutenberg), Wikipedia, academic papers (arXiv, PubMed), code (GitHub), and high-quality filtered web content.

Raw web data is noisy — spam, duplicate content, low-quality pages, toxic content, personally identifiable information. Significant engineering effort goes into filtering: deduplication (removing near-duplicate documents), quality filtering (heuristics and classifiers to remove low-quality text), safety filtering (removing CSAM, extreme content), and mixing ratios (balancing code vs natural language vs books).

The data mix matters enormously for the resulting model's capabilities. More code → better coding. More multilingual text → better non-English performance. More scientific text → better reasoning. Data curation is a competitive moat for frontier labs.

Phase 2: Pre-Training

The Next-Token Prediction Task

The training objective is simple: given all preceding tokens, predict the next one. Given "The capital of France is", the model should assign high probability to "Paris". This is a self-supervised task — the labels are the text itself, requiring no human annotation. A trillion tokens of text provides a trillion training examples.

During pre-training, the model processes mini-batches of text sequences. Each forward pass produces loss across all token positions. Backpropagation adjusts all ~100B–1T parameters to reduce this loss. With a typical learning rate schedule (warmup then cosine decay), this runs for weeks to months on thousands of GPUs.

Phase 3: Supervised Fine-Tuning (SFT)

From Text Predictor to Instruction Follower

After pre-training, the model can continue any text — but it isn't an assistant. If you ask "What is the capital of France?", it might continue with "What is the capital of Germany? What is the capital of Spain?" (continuing the pattern of geographic questions) rather than answering.

SFT fixes this. Human contractors write high-quality demonstration data: (instruction, ideal response) pairs covering thousands of tasks. "Summarise this article:" → "[clean summary]". "Write Python code to sort a list:" → "[working code with explanation]". "I'm feeling sad:" → "[empathetic, helpful response]".

The model is fine-tuned on these examples using standard supervised learning. After SFT, the model follows instructions reliably — but may still sometimes refuse reasonable requests, be inconsistent about content policies, or produce suboptimal responses that the next phase addresses.

Phase 4: RLHF — Reinforcement Learning from Human Feedback

1

Collect Comparison Data

For each of thousands of prompts, the SFT model generates multiple responses (usually 4–9). Human raters compare pairs: "Which of these two responses is better?" This comparison task is much easier and more reliable than writing ideal responses from scratch — humans are better at judging than generating.

Rater guidelines cover: helpfulness, accuracy, safety, harmlessness, tone, and instruction-following. Different labs weight these dimensions differently.

2

Train a Reward Model (RM)

A separate neural network (typically a smaller version of the LLM) is trained to predict human preference scores. Input: (prompt, response). Output: scalar reward score. Trained on the comparison data to give higher scores to responses humans preferred.

The reward model generalises from rated examples to novel prompts and responses — it becomes a proxy for human preference, scalably applying human judgment to any response.

3

Optimise LLM Against Reward Model (PPO)

Using Proximal Policy Optimisation (a reinforcement learning algorithm), the LLM is updated to generate responses that maximise the reward model's score. The LLM is the "policy"; generating responses is the "action"; the reward model provides the "reward signal".

A KL-divergence penalty prevents the LLM from drifting too far from the SFT model (preventing reward hacking — producing gibberish that scores high on the reward model but is useless in practice).

4

Iterate

The process repeats: collect more comparisons on the updated model, retrain the reward model, run more PPO. Each iteration refines the model's alignment with human preferences. Major labs run multiple RLHF iterations before release.

The final result: a model that is measurably more helpful, more honest, less harmful, and better at following instructions than the SFT baseline — quantified by human preference studies.

Alternative: Constitutional AI (Anthropic's Approach)

Scaling Alignment Without Unlimited Human Labels

RLHF requires expensive human comparison data. Anthropic's Constitutional AI (CAI) approach reduces this bottleneck. A written "constitution" — a set of principles the model should follow — replaces some human preference labelling.

Step 1 — Critique and Revision: The model generates an initial response, then critiques it against each constitutional principle ("Does this response assist with illegal activity? If so, rewrite it to refuse helpfully."). The model revises its own output. This generates a large dataset of (harmful initial response, safe revised response) pairs automatically.

Step 2 — AI Feedback: Rather than human raters comparing responses, another AI model (guided by the constitution) provides the preference rankings used to train the reward model. This "RLAIF" (Reinforcement Learning from AI Feedback) scales more cheaply than pure human labelling.

The result is more transparent (the constitution is inspectable), more consistent (AI raters don't fatigue), and more scalable. Claude's helpful, harmless, and honest character is the result of this methodology applied through multiple training iterations.

The Compute Stack

🖥️

GPUs / TPUs

NVIDIA A100s and H100s (GPUs) or Google TPU v4/v5 pods process matrix multiplications at scale. A frontier training run uses thousands simultaneously.

🔗

High-Speed Interconnects

GPUs communicate over NVLink (intra-node) and InfiniBand/RoCE (inter-node). Gradient synchronisation requires enormous bandwidth — hundreds of GB/s between cards.

⚡

Mixed Precision

Weights stored in float16 or bfloat16 (not float32) to halve memory usage and increase throughput. Master weights kept in float32 to prevent precision loss during updates.

🔀

3D Parallelism

Training uses data parallelism (same model, different data batches), tensor parallelism (split model layers across GPUs), and pipeline parallelism (split layers across stages) simultaneously.

Deep Dive — Inference

✨ How LLMs Generate Text

What actually happens inside the model from the moment you press Send to when the first word appears? The mechanics of inference, sampling, and the autoregressive loop.

The Autoregressive Loop

Text generation is autoregressive — the model generates one token at a time, and each new token is appended to the context before the next token is generated. There's no "planning ahead" — the model doesn't decide what to say and then write it. It produces one piece at a time, with each piece influencing all subsequent pieces.

1

Tokenise the Input

Your prompt is tokenised into a sequence of integers. "What is the capital of France?" → [2061, 374, 279, 6864, 315, 9822, 30]. These IDs enter the model.

2

Full Forward Pass Through All Layers

The entire input sequence is processed through all transformer layers simultaneously. For the last token position, the model produces a 50,000-dimensional vector (one value per vocabulary token). After softmax, this is a probability distribution: P("Paris" | context) = 0.87, P("London" | context) = 0.02, etc.

3

Sample the Next Token

With the probability distribution, the model samples the next token. The sampling strategy (controlled by temperature, top-p, top-k) determines how it samples. Temperature=0: always pick the highest probability token ("Paris"). Temperature=1: sample proportionally. Temperature=1.5: flatten the distribution, giving unusual tokens more chance.

4

Append and Repeat

The sampled token is appended to the context sequence. The model now runs another full forward pass on this extended sequence, producing a distribution for the next token. This repeats until a special end-of-sequence token is generated, a maximum length is reached, or a stopping condition is met.

For a 500-token response, this loop runs 500 times. At 100 tokens/second, that's a 5-second wait — why streaming responses are so important for UX.

Key-Value Cache: Speeding Up Inference

Why Repeat Computation? Cache It.

In the autoregressive loop, the same prompt tokens are re-processed in every generation step. For a 1,000-token prompt followed by 500 generated tokens, that's recomputing the same 1,000-token context 500 times — enormously wasteful.

The KV cache (Key-Value cache) solves this. After the initial forward pass, the Key and Value vectors computed in each attention layer for each prompt token are stored (cached). Subsequent generation steps only compute K and V for the newly added token, looking up cached values for all previous tokens. This reduces per-step compute from O(context_length) to O(1), making generation ~5-10x faster on long contexts.

The trade-off: KV cache memory grows linearly with context length and batch size. For a 200K token context with a 70B model, the KV cache alone can require dozens of GB of GPU memory — a key constraint in deploying large-context models.

Sampling Strategies Compared

Strategy

Greedy Decoding (Temp=0)

Always pick the highest-probability token. Fast, deterministic, reproducible. But can get stuck in repetitive loops. Good for factual tasks; bad for creative writing.

Strategy

Temperature Sampling

Sample from the probability distribution scaled by temperature. Temp<1 = conservative; Temp>1 = creative. The standard approach for most use cases.

Strategy

Top-k Sampling

Only consider the k highest-probability tokens. k=40 means only the 40 most likely tokens are eligible. Prevents sampling from very low-probability tail — prevents extremely unlikely outputs.

Strategy

Nucleus (Top-p) Sampling

Include tokens until cumulative probability reaches p (e.g., 0.9). Dynamically adjusts: when model is confident (few tokens have most probability), fewer candidates; when uncertain, more. Best overall sampling strategy for most tasks.

Strategy

Beam Search

Track multiple candidate sequences simultaneously, expanding the best ones. Produces more globally coherent sequences. Used in machine translation; less common for long-form generation where greedy/sampling works better.

Strategy

Repetition Penalty

Reduce the probability of tokens that have already appeared in the output. Prevents boring, repetitive text. Can be too aggressive if the model needs to reference something it already said.

Structured Output

Constrained Decoding

For applications requiring specific output formats (valid JSON, Python code, specific schema), constrained decoding modifies the sampling process to only allow tokens that keep the output in a valid state according to a grammar or schema. At each step, a parser checks what tokens could legally follow the current partial output; only those tokens are candidates for sampling.

This approach guarantees structural validity (the output will always be valid JSON, for example) while allowing the model to fill in the content freely within those structural constraints. Used in production systems where reliability of output format is critical.

Deep Dive — Theory

📈 Scaling Laws

Why bigger models are generally better — the mathematical relationship between model size, data, compute, and capability.

📖 This section is based on Scaling Laws for Neural Language Models (Kaplan et al., 2020) and Training Compute-Optimal LLMs (Hoffmann et al., "Chinchilla", 2022).

The Empirical Power Laws

Kaplan et al. discovered that LLM performance (measured by test loss, lower = better) follows smooth power laws with model size (N parameters), dataset size (D tokens), and compute budget (C FLOPs). These are not step-function improvements or random — they are remarkably smooth and predictable.

Loss(N) ∝ N^(-0.076) — doubling model size reduces loss by ~5%. Loss(D) ∝ D^(-0.095) — doubling data reduces loss by ~6%. Loss(C) ∝ C^(-0.050) — doubling compute reduces loss by ~3%.

The key insight: you can predict how good a larger model will be before training it. This enabled systematic scaling — labs could chart improvement curves and plan multi-year compute investments with confidence.

The Chinchilla Lesson: Data Matters as Much as Size

Compute-Optimal Training

The original Kaplan scaling laws implied that, for a fixed compute budget, you should train the largest possible model for the fewest steps. This led to GPT-3 (175B parameters) being trained on only 300B tokens — potentially underfit for its size.

DeepMind's 2022 Chinchilla paper revised this. For compute-optimal training, model size and training tokens should scale equally. For every doubling of model parameters, training data should also double. Chinchilla (70B parameters, 1.4T tokens) outperformed Gopher (280B parameters, 300B tokens) — 4x fewer parameters but 5x more data.

This reshaped the field. Llama models, Mistral, and others adopted compute-optimal training ratios, producing smaller models with better performance per parameter — and crucially, cheaper to run at inference time.

Emergent Abilities — The Unexpected Surprise

Capabilities That Appear With Scale

Beyond smooth power-law improvements, certain capabilities appear abruptly at scale thresholds — they are essentially absent in smaller models and present in larger ones. These "emergent abilities" were documented in a 2022 survey across dozens of tasks:

In-context learning (few-shot): Following examples in the prompt without gradient updates. Not present at small scale; appears strongly around 10B+ parameters.

Chain-of-thought reasoning: Breaking down problems step by step. Appears around 100B parameters; dramatically improves with scale.

Instruction following: Following novel, complex instructions without fine-tuning. Scales non-linearly.

The mechanism behind emergence is debated — some argue it's an artefact of how capabilities are measured; others argue it reflects genuine phase transitions in learned representations. Either way, the practical implication is clear: scale unlocks qualitatively new capabilities, not just quantitatively better performance.

Deep Dive — Taxonomy

📦 The AI Model Landscape

Not all AI is the same. This page maps the full landscape of model types — what they do, how they work, and when to use each.

Language

💬 Large Language Models

Decoder-only transformers trained on text. Generate text autoregressively. Power chatbots, code assistants, document analysis, summarisation. Examples: GPT-4, Claude, Gemini, Llama 3, Mistral.

Embedding

📐 Embedding Models

Encoder-only models that convert text to dense vectors. Used for semantic search, RAG retrieval, clustering. Don't generate text. Examples: text-embedding-3, E5-large, GTE.

Vision

🖼️ Image Generation — Diffusion

Learn to reverse a noise-adding process. Start from random noise; iteratively denoise to produce images matching a text prompt. Examples: Stable Diffusion, DALL-E 3, Midjourney, Flux.

Vision

👁️ Vision Models

CNN or ViT-based classifiers and detectors. Classify images (ImageNet), detect objects (YOLO), segment pixels (SAM). Foundational in medical imaging, autonomous driving, security.

Multimodal

🎨 Vision-Language Models

Combine image and text understanding in one model. Process images + text together. Examples: GPT-4V, Claude 3, Gemini, LLaVA. Can describe images, answer visual questions, read charts.

Audio

🎵 Speech & Audio Models

Whisper (STT: audio → text), ElevenLabs (TTS: text → speech), Suno/Udio (music generation). Audio waveforms processed as spectrogram patches through transformer-like architectures.

Reasoning

🧮 Reasoning / Thinking Models

Trained to produce long internal reasoning traces before answering. Trade latency for accuracy. Significantly outperform standard LLMs on maths, coding, science. Examples: o1, o3, DeepSeek R1.

Science

🧬 Scientific AI

Domain-specific models for science. AlphaFold 2 predicts protein 3D structure (Nobel Prize impact). AlphaCode competes at competitive programming. These combine domain expertise with LLM scale.

Agents

🤖 Agent Frameworks

Not a model type but an orchestration layer. LLMs augmented with tools (code execution, web search, APIs), planning, and multi-step execution. Claude Code, AutoGPT, and similar systems.

RL

🎮 Reinforcement Learning Models

Learn optimal actions in environments through reward signals. AlphaGo, AlphaZero (games), OpenAI Five (Dota 2). Separate from LLMs but RL is used in RLHF for LLM alignment.

🚀 The trend in 2024–2025 is unification: frontier models like GPT-4o, Claude 3, and Gemini 1.5 are multimodal by design — natively processing text, images, audio, and video in a single model. Specialised single-modality models still excel in their specific domains, but general-purpose frontier models are closing the gap rapidly.
