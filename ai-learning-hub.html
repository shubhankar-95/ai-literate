Section 01

AI Terminology

Hover any card to expand the full explanation. Search to find a specific term.

🔍
Foundation
🤖 Artificial Intelligence (AI)
Machines that mimic human thinking.
AI is the broad field of making computers perform tasks that normally require human intelligence — like understanding language, recognising images, or making decisions. Think of it as the parent category for everything else on this page.
📖 IBM — What is AI?
Foundation
📊 Machine Learning (ML)
Teaching machines by showing them examples.
Instead of manually writing every rule, you feed the computer thousands of examples and it learns patterns on its own. A spam filter learns what spam looks like by studying millions of emails — nobody wrote explicit rules.
📖 Google — ML Crash Course
Foundation
🕸️ Deep Learning (DL)
ML using brain-inspired layered networks.
A subfield of ML that uses Neural Networks with many layers ("deep"). Each layer learns progressively more abstract features. Layer 1 might learn edges in an image; layer 10 might recognise faces. This powers image recognition, voice assistants, and LLMs.
📖 DeepLearning.AI — Courses
Key Concept
💬 Large Language Model (LLM)
AI trained on massive text to understand & generate language.
An LLM is trained on billions of words from the internet, books, and code. It learns statistical patterns so well that it can write essays, answer questions, translate languages, and write software. GPT-4, Claude, and Gemini are all LLMs. "Large" refers to the enormous number of parameters (weights).
📖 arXiv — A Survey of LLMs
Architecture
⚡ Transformer
The architecture behind modern AI.
Introduced in the 2017 paper "Attention Is All You Need", the Transformer is the neural network design that powers nearly every modern AI. Its key innovation is the "attention mechanism" — a way for the model to consider every word in context of every other word simultaneously, rather than reading left-to-right sequentially.
📖 Vaswani et al. (2017) — Attention Is All You Need
Core
🧩 Token
The basic unit of text an AI reads.
LLMs don't read characters or whole words — they read tokens. A token is roughly 3-4 characters or ¾ of a word. "ChatGPT" = 2 tokens; "Hello, how are you?" ≈ 6 tokens. Pricing and context limits are measured in tokens. Think of them as pieces of a word-puzzle that the AI reassembles.
📖 OpenAI Tokenizer — Try it live
Core
🗂️ Context Window
How much text an AI can "see" at once.
The context window is the maximum number of tokens an AI can process in one conversation — its working memory. If a model has a 200K token context window, it can read roughly 150,000 words at once. Older models had 4K tokens; Claude 3.5 has 200K. Information outside the window is "forgotten".
📖 Anthropic — Claude Model Overview
Technical
🔢 Parameters
The "knobs" inside a model that store knowledge.
Parameters are numerical values inside the neural network that get adjusted during training. A model with more parameters can, in theory, store more knowledge and handle more complex tasks. GPT-3 had 175 billion parameters. Numbers like "70B" or "7B" in model names refer to parameter count. More parameters = bigger model = more compute needed.
📖 HuggingFace — Transformers Docs
Process
🏋️ Training
The process of teaching a model on data.
Training means feeding the model massive amounts of text and repeatedly adjusting its parameters to minimise prediction errors. Pre-training is the initial phase on broad internet data. It takes thousands of GPUs running for months and costs tens of millions of dollars. Once trained, the model's weights are "frozen" (fixed).
📖 Coursera — Generative AI with LLMs
Process
🎯 Fine-Tuning
Extra training to specialise a general model.
After pre-training, models are fine-tuned on curated, task-specific datasets. For chat models, this includes RLHF (Reinforcement Learning from Human Feedback) — where human raters score responses, and the model learns to generate preferred answers. This is why ChatGPT gives helpful, polite answers rather than raw internet-scraped text.
📖 Ouyang et al. (2022) — InstructGPT (RLHF)
Process
⚙️ Inference
Running a trained model to get outputs.
Inference is what happens when you type a message and the AI responds — the model uses its trained weights to generate output. This is different from training. Inference is much cheaper, but still requires significant GPU compute for large models. Many companies use specialised inference hardware (like NVIDIA A100s) to keep response times fast.
Interaction
✍️ Prompt
The instruction you give an AI.
A prompt is the text input you send to an AI model. The quality of your prompt dramatically affects output quality. "Write a story" gives mediocre results. "Write a 500-word tense thriller set in Tokyo, told in first person, with a plot twist in the final paragraph" gives great results. Prompt engineering is the art of crafting effective prompts.
📖 Anthropic — Prompt Engineering Guide
Parameter
🌡️ Temperature
Controls how random/creative the AI is.
Temperature (0.0 – 2.0) controls how deterministic the output is. At temperature=0 the model always picks the most likely next token (predictable, factual). At temperature=1.5+ it becomes wild and creative (more randomness). For coding tasks, use low temperature. For brainstorming or creative writing, use higher temperature.
Limitation
👻 Hallucination
When an AI confidently makes things up.
LLMs sometimes generate plausible-sounding but factually incorrect information — fake citations, wrong dates, invented people. This isn't lying; the model doesn't have intent. It's a statistical artefact: the model generates the most likely-sounding continuation, which isn't always the true one. Always verify critical facts from AI outputs.
📖 Ji et al. — Survey of Hallucination in NLG
Technical
📐 Embeddings
Turning text into numbers that capture meaning.
An embedding converts a word, sentence, or document into a list of numbers (a vector) where similar meanings end up close together in vector space. "king" - "man" + "woman" ≈ "queen". Embeddings power semantic search, recommendation systems, and RAG (Retrieval-Augmented Generation).
📖 OpenAI — Embeddings Guide
Technique
🔍 RAG (Retrieval-Augmented Generation)
Giving an AI fresh info before it answers.
RAG solves the "outdated knowledge" and hallucination problem. Before answering, the system searches a database (like company docs) for relevant passages and injects them into the prompt as context. The model then answers grounded in that retrieved information, not just its training data. Used heavily in enterprise AI chatbots.
📖 Lewis et al. (2020) — RAG Paper
Capability
🎨 Multimodal AI
AI that understands both text AND images/audio.
A multimodal model can process multiple types of input — text, images, audio, video, and even code. GPT-4o, Claude 3, and Gemini Ultra are multimodal. You can send a photo of a receipt and ask the AI to extract totals, or share a chart and ask for analysis. This dramatically expands what AI can do.
Advanced
🤝 AI Agent
An AI that can take actions, not just answer.
An AI agent doesn't just answer questions — it takes actions: browsing the web, writing and running code, managing files, calling APIs, and completing multi-step goals. It plans, acts, observes results, and tries again. Claude Code is an AI agent for software development. Agents are the frontier of current AI capabilities.
📖 Anthropic — Building Effective Agents
Field
🗣️ NLP (Natural Language Processing)
The AI field focused on human language.
NLP is the subfield of AI dedicated to understanding and generating human language. Classic NLP tasks: sentiment analysis, named entity recognition, machine translation. LLMs essentially solved many NLP tasks by training on enough text data. ChatGPT, Claude, and Gemini are all NLP systems at their core.
Infrastructure
🖥️ GPU & Compute
The specialised hardware that powers AI.
Graphics Processing Units (GPUs) — originally for gaming — are ideal for AI because they can perform thousands of parallel calculations. Training large models requires clusters of thousands of GPUs running for months (costing $50M+). NVIDIA dominates the AI chip market. Cloud providers (AWS, Google, Azure) rent GPU time by the hour.
Prompting
⚙️ System Prompt
Hidden instructions that set the AI's behaviour.
A system prompt is a set of instructions given to the AI before the conversation starts — setting its persona, rules, and context. "You are a customer support agent for Acme Corp. Only answer questions about our products. Be concise." Users usually don't see system prompts, but they heavily influence model behaviour.
📖 Anthropic — System Prompts
Ecosystem
🔓 Open-Source Models
AI models anyone can download and run.
Unlike closed models (GPT-4, Claude), open-source models like Meta's Llama 3 and Mistral release their weights publicly. Anyone can download, run, and fine-tune them. Advantages: privacy, customisation, no API costs. Disadvantages: requires hardware to run, typically less capable than frontier closed models. HuggingFace hosts thousands of open models.
📖 HuggingFace — Model Hub

Section 02

Core AI Components & Models

How the technology actually works, explained simply.

💡 The Transformer architecture (2017) is the backbone of virtually every modern AI. Understanding it gives you a mental model of why these systems work.
1

Tokenisation — Splitting text into pieces

Your input sentence is first broken into tokens. "The cat sat" → ["The", " cat", " sat"]. Each token is mapped to a unique number ID. This is the raw input the model receives.

2

Embeddings — Converting IDs to meaning vectors

Each token ID is looked up in an embedding table, producing a dense numerical vector (e.g. 4096 numbers). This vector encodes the word's semantic meaning in a high-dimensional space.

3

Positional Encoding — Adding word order

Since Transformers process all tokens simultaneously (unlike RNNs), positional information is injected into each embedding — telling the model "this token is at position 5."

4

Self-Attention — Every word sees every other word

This is the magic. Each token computes "how relevant is every other token to me?" For the word "bank" in "I went to the river bank", attention weights tell the model to focus on "river" not "money". This creates context-aware representations.

5

Feed-Forward Layers — Transforming representations

After attention, each token's representation passes through a feed-forward network (basically two linear transformations with a non-linearity). This adds capacity for complex computation.

6

Stacking Layers — Going "Deep"

Steps 4–5 repeat 32, 80, or even 128 times (the "depth"). Each layer refines understanding. Lower layers learn syntax; higher layers learn semantics and reasoning. GPT-4 reportedly has ~120 layers.

7

Output — Predicting the next token

The final layer produces probability scores for every possible next token. The model samples from this distribution (influenced by temperature). It then outputs one token, appends it to the input, and repeats — this is called "autoregressive" generation.

📖 Reference: Vaswani et al., "Attention Is All You Need" (2017) — The original Transformer paper with only 8 authors that changed AI forever.
Basics

🧠 What is a Neural Network?

Inspired by the brain, a neural network is layers of interconnected "neurons". Each connection has a weight (a number). Input passes through layers, gets multiplied by weights, transformed, and produces output. Training = finding the right weights.

Learning

📉 Backpropagation

After a forward pass, the model calculates how wrong its prediction was (loss). Backpropagation computes how much each weight contributed to the error and adjusts it via gradient descent. Repeat millions of times = learning.

Design

🔁 Attention Mechanism

Attention lets any part of the network directly access any other part. It computes three matrices — Query (what am I looking for?), Key (what do I contain?), Value (what information do I hold?) — and combines them. Multi-head attention does this in parallel with different "views".

Scale

📈 Scaling Laws

Kaplan et al. (2020) showed that model capability improves predictably with more parameters, more data, and more compute — following smooth power laws. This insight drove the "race to scale" and the creation of GPT-3, GPT-4, and their successors.

📖 Scaling Laws for Neural Language Models (2020)
1

Pre-training — Reading the internet

The model sees trillions of tokens from web pages, books, Wikipedia, GitHub, and more. It learns to predict the next word. Simple task, profound result: to predict well, it must learn grammar, facts, reasoning, and code. This phase is the most expensive — weeks on 10,000+ GPUs.

2

Supervised Fine-Tuning (SFT)

Human contractors write ideal question-answer pairs. The model is fine-tuned on these examples to follow instructions. This is where it transitions from "predict the next word" to "be a helpful assistant". Requires far less data and compute than pre-training.

3

RLHF — Learning from human preferences

Human raters compare two model outputs and pick the better one. A "reward model" is trained to predict these preferences. The LLM is then fine-tuned using RL to maximise the reward model's score. This makes outputs more helpful, harmless, and honest.

📖 InstructGPT Paper — RLHF for LLMs
4

Constitutional AI (Claude's approach)

Anthropic's technique for Claude: instead of only human feedback, a set of AI principles ("constitution") guides model behaviour. The model critiques its own outputs and revises them, scaling alignment more efficiently. Enables safer, more consistent behaviour.

📖 Anthropic — Constitutional AI (2022)
Text

📝 Language Models (LLMs)

Trained to understand and generate text. Examples: GPT-4, Claude 3, Gemini, Llama 3. Powers chatbots, coding assistants, document analysis, summarisation, and more.

Image

🎨 Diffusion Models

Generate images by learning to "denoise" random noise. Examples: Stable Diffusion, DALL-E 3, Midjourney. They start with noise and iteratively remove it to reveal a coherent image matching your prompt.

Audio

🎵 Audio Models

Whisper (OpenAI) transcribes speech. ElevenLabs and Suno generate realistic voices and music. These models are trained on audio waveforms rather than text tokens, but similar transformer principles apply.

Code

💻 Code Models

Trained heavily on GitHub repositories. Examples: GitHub Copilot, Claude for code, Codestral. Can write, review, debug, and explain code in dozens of programming languages. Often faster and more accurate than a general LLM for pure coding tasks.

Vision

👁️ Vision Models

Classify images (ResNet, Vision Transformer), detect objects (YOLO), or segment images (SAM by Meta). LLMs like GPT-4V and Claude 3 combine language + vision in a single multimodal model.

Reasoning

🧮 Reasoning Models

Models optimised to "think before answering" using chain-of-thought and extended inference. Examples: OpenAI o1/o3, DeepSeek R1. Trade response speed for accuracy on complex maths and logic problems.

For coding & long documents

Use Claude 3.5 Sonnet — best instruction-following, longest context, excellent at reading entire codebases or lengthy PDFs.

For everyday chatting & images

Use GPT-4o — most polished UX, built-in image generation with DALL-E, voice mode, and the widest tool/plugin ecosystem.

Use Gemini 1.5 Pro — native integration with YouTube, Google Drive, Gmail. Unmatched context for multimedia.

Use OpenAI o1 or o3 — slower but far more accurate on problems requiring multi-step logical reasoning.

You need Node.js 18+ and an Anthropic API key. Get one free at console.anthropic.com

Terminal

Terminal

# Add to ~/.bashrc or ~/.zshrc to make it permanent

Claude Code now has access to your project files. Ask it anything about the code or give it tasks.

Section 05

Prompting Guidelines

The quality of your prompt determines the quality of your output. These techniques work across ChatGPT, Claude, and Gemini.

🎭

Give it a role

Start with "You are a senior Python developer…" or "You are a UX designer…". Roles activate domain-specific knowledge and tone.

🎯

Be specific about the output

Specify format, length, tone, audience. "Write a 3-sentence summary for a non-technical CEO" is better than "summarise this".

📚

Provide context

Include relevant background. The more context the AI has, the more tailored its answer. Paste in error messages, code snippets, or relevant documents.

✅

Show examples (few-shot)

Give 1–3 examples of your desired input/output format before your actual request. This is the single most reliable way to control output format.

🔢

Ask for step-by-step reasoning

Add "Think step by step" or "Work through this carefully before giving the final answer". This dramatically improves accuracy on complex tasks.

🔄

Iterate, don't restart

Start simple, see the output, then refine: "Make it shorter", "Add more technical detail", "Change the tone to casual". Prompting is a conversation.

Technique

Zero-Shot Prompting

Ask directly with no examples. Works for simple, common tasks. "Translate this to Spanish: Hello, world." Best for straightforward requests where the task is well-understood.

Technique

Few-Shot Prompting

Provide 2-5 examples of input → output before your actual request. Dramatically improves consistency. Used when you want a very specific output format the model might not default to.

Technique

Chain-of-Thought (CoT)

Ask the model to reason step-by-step before answering. "Let's think through this step by step." Shown to dramatically improve maths, logic, and multi-step reasoning accuracy.

📖 Wei et al. (2022) — Chain-of-Thought Prompting
Technique

Role / Persona Prompting

Assign a specific identity: "You are an expert SQL database administrator with 15 years of experience optimising PostgreSQL." Strong personas improve depth and accuracy in specialised domains.

Technique

Structured Output Prompting

Specify exact output format: "Return your answer as a JSON object with keys: title, summary, tags, difficulty_level." Useful for integrating AI output into code or spreadsheets.

Technique

Self-Consistency

Ask the model to solve the same problem multiple times and pick the most common answer, or ask it to critique its own output: "Now review your answer and identify any errors."

❌ Weak vs ✅ Strong Prompts

Example 1 — Summarisation

❌ Weak Prompt
Summarise this article.
✅ Strong Prompt
Summarise the following article in exactly 3 bullet points. Each bullet should be one sentence. Write for a busy executive who has 30 seconds to read this. Focus on business impact, not technical details.

[Article text here]

Example 2 — Code Generation

❌ Weak Prompt
Write a login function in Python.
✅ Strong Prompt
You are a senior Python backend developer. Write a login() function for a FastAPI app that:
- Accepts email (str) and password (str) as parameters
- Queries a PostgreSQL database via SQLAlchemy ORM
- Uses bcrypt for password hashing
- Returns a JWT token on success, raises HTTPException 401 on failure
- Includes docstring and type hints
Do not use any external auth libraries. Include example usage in a comment.

Example 3 — System Prompt + User Turn

⚙️ System Prompt
You are Alex, a friendly customer support agent for TechFlow SaaS. You help users with billing, account issues, and basic troubleshooting. Be warm, concise, and always offer to escalate to a human if you can't resolve the issue. Never make up pricing — if you don't know, say so.
👤 User
I was charged twice this month!
🤖 Claude (as Alex)
Hi! I'm so sorry to hear that — a double charge is definitely something we want to fix right away. Could you share the email address on your account? I'll pull up your billing history and get this sorted for you. If needed, I can escalate to our billing team to process a refund same-day.
📖 Official guides: Anthropic Prompt Engineering · Prompt Engineering Guide (community)

🚫 Being too vague

"Help me with my business" → the AI has no idea what you need. Always specify the task, format, and audience.

🚫 Asking too many things at once

10 different questions in one prompt dilutes focus. Break complex tasks into separate, focused prompts or use numbered steps.

🚫 Not providing context

"Fix this code" without sharing the code. Or "write about our company" without describing what the company does.

🚫 Treating first output as final

The best outputs come from iteration. Always refine: "Make it shorter / more formal / add an example / fix the third point."

🚫 Blindly trusting facts

LLMs hallucinate. Never trust names, dates, citations, statistics, or legal/medical claims without verifying from a primary source.

🚫 Not specifying output format

Without format instructions, AI defaults to verbose paragraphs. Specify: "Return as a numbered list", "Use markdown headers", "JSON with these fields".
